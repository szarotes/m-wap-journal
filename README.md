# m-wap-journal

## 项目简介

`m-wap-journal` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，旨在以结构化的方式保存和展示一系列静态页面，便于长期维护与访问。

## 目录说明

```
m-wap-journal/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面依赖的静态资源（CSS、JS、图片等）
├── index.html      # 项目主页（可选）
└── README.md       # 本文件
```

- **pages/**：核心目录，每个子文件夹或文件对应一个独立的归档页面。命名建议使用日期或编号，以保持清晰。
- **assets/**：页面所需的样式、脚本、字体、图片等资源，按需组织子目录。
- **index.html**：如存在，可作为导航或入口页面。

## 页面归档说明

- 每个 HTML 页面应是完整自包含的，或通过相对路径引用 `assets/` 下的资源。
- 页面内容应保持原始风貌，不进行额外修改，仅作归档用途。
- 归档时请遵循仓库内约定的命名规范（如 `YYYY-MM-DD-title.html` 格式），具体规范可参考仓库内其他文件。

## 维护说明

- 欢迎提交 Pull Request 添加新的归档页面，或修正现有资源链接。
- 请勿在页面中插入任何形式的追踪代码、广告或外部请求（除非必要且注明）。
- 本仓库仅用于技术归档，不承担任何页面内容的审核责任，亦不代表对页面内容的态度。
- 如有争议内容，仓库维护者保留移除或修改的权利。

## 注意事项

- 本仓库为静态页面集合，不涉及后端服务或动态内容。
- 所有页面仅供学习、存档或演示用途。
- 使用本仓库时请遵守相关法律法规及开源协议。
